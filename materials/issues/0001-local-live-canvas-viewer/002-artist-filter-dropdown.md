## Что строим

Поверх MVP live-вьюера — UI-фильтр по `artist_name`, чтобы артист мог наблюдать пульс только выбранных соседей.

Сценарий end-to-end:

1. В верхней части страницы — кнопка-триггер фильтра.
2. По клику на кнопку разворачивается dropdown со списком чекбоксов: один чекбокс на каждого артиста, чьи штрихи приходили по WS с момента загрузки страницы.
3. Каждый раз, когда renderer видит новый `artist_name`, он добавляется в список с `enabled=true` (галочка стоит) — артист сразу виден в пульсе.
4. Клик по чекбоксу переключает `enabled` для соответствующего артиста. Снятая галочка означает: штрихи этого артиста не рисуются ни в текущем, ни в последующих кадрах. WS-соединение остаётся живым.
5. Состояние фильтра не сохраняется между перезагрузками (см. «За рамками» в PRD).

Внутри одного `<script>` это два логических модуля:

- **`artistFilter`** — держит `Map<artist_name, enabled: bool>`, отвечает на `observe(artist_name)`, `isEnabled(artist_name)`, `reset()`, эмитит изменения наружу.
- **`dropdown`** — кнопка + раскрывающийся список, подписан на изменения `artistFilter`, рендерит чекбоксы, по клику дёргает переключение состояния.

Renderer на каждом тике 30fps дополнительно фильтрует штрихи через `artistFilter.isEnabled(stroke.artist_name)` перед отрисовкой.

`reset()` будет использован в issue про `clear` — здесь достаточно того, что метод существует и обнуляет состояние/UI.

## Критерии выполненной задачи

- [ ] Кнопка-триггер фильтра видна в верхней части страницы.
- [ ] Клик по кнопке разворачивает / сворачивает dropdown со списком артистов.
- [ ] Новый `artist_name`, чьи штрихи только что пришли по WS, автоматически появляется в dropdown с включённой галочкой и его штрихи сразу рисуются.
- [ ] Снятие галочки убирает штрихи этого артиста из текущего и всех последующих кадров; WS-соединение не разрывается.
- [ ] Возврат галочки восстанавливает отрисовку штрихов артиста на последующих кадрах.
- [ ] Состояние фильтра в памяти страницы, не персистится (между перезагрузками — сбрасывается естественным образом).
- [ ] Тесты на UI-фильтр согласно «Решения по тестированию» родительского PRD: Playwright-сценарий с двумя уникальными артистами через `POST /canvas/register`, посылка штрихов от обоих, снятие галочки у одного через `browser_click`, посылка новых штрихов, проверка через `browser_evaluate`/`getImageData` что только включённый артист отрисован.
